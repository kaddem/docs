# Правила написания CSS

## Синтаксис

1. Селекторы, объявления и значения пиши строчными буквами:
```css
/* Плохо: */
.Selector {
  WIDTH: 50PX;
}

/* Хорошо: */
.selector {
  width: 50px;
}
```
<br>

2. Ставь пробел после селектора и до открывающей фигурной скобки. После открывающей фигурной скобки делай перенос строки. Закрывающую фигурную скобку пиши с новой строки и после неё оставляй пустую строку
```css
/* Плохо: */
.selector{width: 50px;}
.next-selector{width: 50px;}

/* Хорошо: */
.selector {
  width: 50px;
}

.next-selector {
  width: 50px;
}
```
<br>

3. Группируя селекторы пиши каждый на отдельной строке:
```css
/* Плохо: */
.selector, .yet-selector {
  width: 50px;
}

/* Хорошо: */
.selector,
.yet-selector {
  width: 50px;
}
```
<br>

4. Пиши каждое объявление свойства на новой строке:
```css
/* Плохо: */
.selector {
  width: 50%; padding: 0 15px;
}

/* Хорошо: */
.selector {
  width: 50%;
  padding: 0 15px;
}
```
<br>

4. Ставь ```:``` сразу после объявления свойства. После ```:``` ставь один пробел:
```css
/* Плохо: */
.selector {
  width :50%;
}

/* Хорошо: */
.selector {
  width: 50%;
}
```
<br>

5. Ставь ```;``` в конце каждого объявления:
```css
/* Плохо: */
.selector {
  width: 50%
}

/* Хорошо: */
.selector {
  width: 50%;
}
```
<br>

6. Опускай единицы измерения для нулевых значений:
```css
/* Плохо: */
.selector {
  margin: 0px;
}

/* Хорошо: */
.selector {
  margin: 0;
}
```
<br>

7. Пиши пробел после запятой внутри таких значений как ```rgb()```, ```rgba()```, ```linear-gradient()``` и подобных:
```css
/* Плохо: */
.selector {
  background-color: rgba(0,0,0,.5);
  background-image: linear-gradient(45deg,#333,#333 50%,#eee 75%,#333 75%);
}

/* Хорошо: */
.selector {
  background-color: rgba(0, 0, 0, .5);
  background-image: linear-gradient(45deg, #333, #333 50%, #eee 75%, #333 75%);
}
```
<br>

8. Опускай ноль для дробных значений:
```css
/* Плохо: */
.selector {
  opacity: 0.5;
}

/* Хорошо: */
.selector {
  opacity: .5;
}
```
<br>

9. HEX значения пиши строчными буквами:
```css
/* Плохо: */
.selector {
  color: #3E3E3E;
}

/* Хорошо: */
.selector {
  color: #3e3e3e;
}
```
<br>

10. Сокращай HEX значения по возможности:
```css
/* Плохо: */
.selector {
  color: #ffffff;
}

/* Хорошо: */
.selector {
  color: #fff;
}
```
<br>

11. Ставь пробелы до и после комбинаторов:
```css
/* Плохо: */
.prev+.next  {
  margin-top: 2em;
}

/* Хорошо: */
.prev + .next  {
  margin-top: 2em;
}
```
<br>

12. Пиши значения атрибутов в кавычках:
```css
/* Плохо: */
a[href^=http] {
  ...
}

/* Хорошо: */
a[href^="http"] {
  ...
}
```
<br>

13. Настрой редактор так, что бы отступы объявлений формировались двумя символами пробела - размести файл [.editorconfig](https://github.com/kaddem/docs/blob/master/.editorconfig) (универсальные настройки редактора) в папке проекта и установи его поддержку в редакторе.
Что бы VSCode научился работать с ```.editorconfig``` файлом - [установи плагин](https://marketplace.visualstudio.com/items?itemName=EditorConfig.EditorConfig)

<br>

## Селекторы

1. Используй классы
```css
/* Плохо: */
nav {
  margin-left: auto;
}

/* Хорошо: */
.main-nav {
  margin-left: auto;
}
```
<br>

2. Избегай тегов в селекторах. Излишняя специфичность (вес) селекторов, может усложнить стилизацию. Используя только классы, специфичность селекторов будет одинаковой - теперь свойство написанное ниже по коду прогнозируемо переопределяет написанное выше.
```css
/* Плохо: */
nav.main-nav {
  margin-left: auto;
}

/* Хорошо: */
.main-nav {
  margin-left: auto;
}
```
<br>

3. Пиши короткие селекторы, избегая составных. Тем не менее, если не использовать методологию именования классов БЭМ (Блок - Элемент - Мдификатор), может понадобиться сделать составной селектор. В этом случае пиши селекор максимально компактно:
```css
/* Плохо: */
.main-nav ul li a {
  color: inherit;
}

/* Допустимо: */
.main-nav a {
  color: inherit;
}

/* Хорошо: */
.main-nav-link {
  color: inherit;
}
```
<br>

4. Пиши селекторы строчными буквами. Используй дефис для разделения слов в селекторе:
```css
/* Плохо: */
.Main_Nav {
  margin-left: auto;
}

/* Хорошо: */
.main-nav {
  margin-left: auto;
}
```
<br>

4. Пиши селекторы английскими словами - транслит запрещен:
```css
/* Плохо: */
.uslugi {
  margin-left: auto;
}

/* Хорошо: */
.services {
  margin-left: auto;
}
```
<br>

5. Придумывай для классов/селекторов короткие и емкие названия. Помни - имя класса должно отвечать на вопрос - "Что это?" и НЕ должно отвечать на вопрос - "Как это выглядит?" (исключение - класс модификатор)
```css
/* Плохо: */
.red-icon-facebook-in-footer {} /* Слишком длинное название */
.icon1 {} /* НЕ информативное название - что это за иконка?
А если мконок пара десятков на весь сайт, можно запуться */
.block {} /* НЕ информативное название */
.red {} /* А что если html тег с таким классом нужно будет сделать зеленым? */

/* Хорошо: */
.main-nav { ... }
.main-nav__item { ... } /* селектор с двойным подчеркиванием - пример БЭМ нотации */
.main-nav__link { ... }
.social-icon { ... }
.comment { ... }
.page-header { ... }
```
Придумывая имена классов обрати внимание на [слова, часто используемые в CSS-классах](https://github.com/yoksel/common-words)
<br><br>

6. Пиши префикс .js-* у классов для работы с JavaScript. Такие классы должны отсутствовать в CSS.
<br><br>

## Блок объявлений
**Блок объявлений** - все что идет за селектором. Это фигурные скобки и сами объявления внутри них.

1. Пиши объявления внутри блока объявлений однотипно в одинаковой последовательности:
    1. Позиционирование
    2. Свойство ```display```
    3. flexbox свойства
    2. Блочная модель и размеры
    3. Свойства текста
    4. Отображение
    5. Остальное
    6. Плавные переходы, анимация
```css
.selector {
  position: absolute;
  top: 0;
  right: 0;
  bottom: 0;
  left: 0;
  z-index: 100;

  display: flex;

  flex-wrap: wrap;
  justify-content: space-between;

  padding: 15px;
  margin-top: 20px;

  font-family: Arial, "Helvetica Neue", sans-serif;
  font-size: 13px;
  line-height: 1.5;
  color: #333;
  text-align: center;

  background-color: #f5f5f5;
  border: 1px solid #e5e5e5;
  border-radius: 3px;
  opacity: 1;

  transition: all .3s;
}
```
<br>

2. Избегай сокращеных записей объявлений:
```css
/* Плохо: */
.selector {
  padding: 0 0 20px;
}

/* Хорошо: */
.selector {
  padding-bottom: 20px;
}
```
