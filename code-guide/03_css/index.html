    Правила написания CSS#

    Синтаксис#

    1. Селекторы, объявления и значения пиши строчными буквами: 

      /* Плохо: */
.Selector {
  WIDTH: 50PX;
}

/* Хорошо: */
.selector {
  width: 50px;
}

    2. Ставь пробел после селектора и до открывающей фигурной скобки. После открывающей фигурной скобки делай перенос строки. Закрывающую фигурную скобку пиши с новой строки и после неё оставляй пустую строку 

      /* Плохо: */
.selector{width: 50px;}
.next-selector{width: 50px;}

/* Хорошо: */
.selector {
  width: 50px;
}

.next-selector {
  width: 50px;
}

    3. Группируя селекторы пиши каждый на отдельной строке: 

      /* Плохо: */
.selector, .yet-selector {
  width: 50px;
}

/* Хорошо: */
.selector,
.yet-selector {
  width: 50px;
}

    4. Пиши каждое объявление свойства на новой строке: 

      /* Плохо: */
.selector {
  width: 50%; padding: 0 15px;
}

/* Хорошо: */
.selector {
  width: 50%;
  padding: 0 15px;
}

    5. Ставь : сразу после объявления свойства. После : ставь один пробел: 

      /* Плохо: */
.selector {
  width :50%;
}

/* Хорошо: */
.selector {
  width: 50%;
}

    6. Ставь ; в конце каждого объявления: 

      /* Плохо: */
.selector {
  width: 50%
}

/* Хорошо: */
.selector {
  width: 50%;
}

    7. Опускай единицы измерения для нулевых значений: 

      /* Плохо: */
.selector {
  margin: 0px;
}

/* Хорошо: */
.selector {
  margin: 0;
}

    8. Пиши пробел после запятой внутри таких значений как rgb(), rgba(), linear-gradient() и подобных: 

      /* Плохо: */
.selector {
  background-color: rgba(0,0,0,.5);
  background-image: linear-gradient(45deg,#333,#333 50%,#eee 75%,#333 75%);
}

/* Хорошо: */
.selector {
  background-color: rgba(0, 0, 0, .5);
  background-image: linear-gradient(45deg, #333, #333 50%, #eee 75%, #333 75%);
}

    9. Опускай ноль для дробных значений: 

      /* Плохо: */
.selector {
  opacity: 0.5;
}

/* Хорошо: */
.selector {
  opacity: .5;
}

    10. HEX значения пиши строчными буквами: 

      /* Плохо: */
.selector {
  color: #3E3E3E;
}

/* Хорошо: */
.selector {
  color: #3e3e3e;
}

    11. Сокращай HEX значения по возможности: 

      /* Плохо: */
.selector {
  color: #ffffff;
}

/* Хорошо: */
.selector {
  color: #fff;
}

    12. Ставь пробелы до и после комбинаторов: 

      /* Плохо: */
.prev+.next  {
  margin-top: 2em;
}

/* Хорошо: */
.prev + .next  {
  margin-top: 2em;
}

    13. Пиши значения атрибутов в кавычках: 

      /* Плохо: */
a[href^=http] {
  ...
}

/* Хорошо: */
a[href^="http"] {
  ...
}

    14. Настрой редактор так, что бы отступы объявлений формировались двумя символами пробела - размести файл .editorconfig (универсальные настройки редактора) в папке проекта и установи его поддержку в редакторе. Что бы VSCode научился работать с .editorconfig файлом - установи плагин


    Селекторы#

    1. Используй классы 

      /* Плохо: */
nav {
  margin-left: auto;
}

/* Хорошо: */
.main-nav {
  margin-left: auto;
}

    2. Избегай тегов в селекторах. Излишняя специфичность (вес) селекторов, может усложнить стилизацию. Используя только классы, специфичность селекторов будет одинаковой - теперь свойство написанное ниже по коду прогнозируемо переопределяет написанное выше. 

      /* Плохо: */
nav.main-nav {
  margin-left: auto;
}

/* Хорошо: */
.main-nav {
  margin-left: auto;
}

    3. Пиши короткие селекторы, избегая составных. Тем не менее, если не использовать методологию именования классов БЭМ (Блок - Элемент - Мдификатор), может понадобиться сделать составной селектор. В этом случае пиши селекор максимально компактно: 

      /* Плохо: */
.main-nav ul li a {
  color: inherit;
}

/* Допустимо: */
.main-nav a {
  color: inherit;
}

/* Хорошо: */
.main-nav-link {
  color: inherit;
}

    4. Пиши селекторы строчными буквами. Используй дефис для разделения слов в селекторе: 

      /* Плохо: */
.Main_Nav {
  margin-left: auto;
}

/* Хорошо: */
.main-nav {
  margin-left: auto;
}

    5. Пиши селекторы английскими словами - транслит запрещен: 

      /* Плохо: */
.uslugi {
  margin-left: auto;
}

/* Хорошо: */
.services {
  margin-left: auto;
}

    6. Придумывай для классов/селекторов короткие и емкие названия. Помни - имя класса должно отвечать на вопрос - "Что это?" и НЕ должно отвечать на вопрос - "Как это выглядит?" (исключение - класс модификатор) 

      /* Плохо: */
.red-icon-facebook-in-footer {} /* Слишком длинное название */
.icon1 {} /* НЕ информативное название - что это за иконка?
А если мконок пара десятков на весь сайт, можно запуться */
.block {} /* НЕ информативное название */
.red {} /* А что если html тег с таким классом нужно будет сделать зеленым? */

/* Хорошо: */
.main-nav { ... }
.main-nav__item { ... } /* селектор с двойным подчеркиванием - пример БЭМ нотации */
.main-nav__link { ... }
.social-icon { ... }
.comment { ... }
.page-header { ... }
      Придумывая имена классов обрати внимание на слова, часто используемые в CSS-классах

    7. Пиши префикс .js-* у классов для работы с JavaScript. Такие классы должны отсутствовать в CSS.

    Блок объявлений#

    Блок объявлений - все что идет за селектором. Это фигурные скобки и сами объявления внутри них.

    1. Пиши объявления внутри блока объявлений однотипно в одинаковой последовательности:

      1. Позиционирование
      2. Свойство display
      3. flexbox свойства
      4. Блочная модель и размеры
      5. Свойства текста
      6. Отображение
      7. Остальное
      8. Плавные переходы, анимация 
        .selector {
  position: absolute;
  top: 0;
  right: 0;
  bottom: 0;
  left: 0;
  z-index: 100;

  display: flex;

  flex-wrap: wrap;
  justify-content: space-between;

  padding: 15px;
  margin-top: 20px;

  font-family: Arial, "Helvetica Neue", sans-serif;
  font-size: 13px;
  line-height: 1.5;
  color: #333;
  text-align: center;

  background-color: #f5f5f5;
  border: 1px solid #e5e5e5;
  border-radius: 3px;
  opacity: 1;

  transition: all .3s;
}

    2. Избегай сокращеных записей объявлений: 

      /* Плохо: */
.selector {
  padding: 0 0 20px;
}

/* Хорошо: */
.selector {
  padding-bottom: 20px;
}
